# raw/projects/

Use this folder for project source material in a private workspace.

## What Could Go Here

- Project briefs.
- Status notes.
- Roadmaps.
- Sanitized artifacts.
- Decision inputs.

## What To Write In The Wiki

- Objective.
- Current state.
- Stakeholders by role, not private name, unless your workspace is private.
- Decisions.
- Risks.
- Evidence links.

## Tradeoff

Raw project files preserve detail, but project pages should carry the reusable synthesis so future agents do not need to reopen every source.
